1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

git checkout название файла (можно с точкой для всех файлов) возвращает на последний коммит.


2.Как вернуть файл из индекса в рабочий каталог?

git restore --staged название файла (файл изменён но не индексирован).


3.Как исправить, отменить  ошибочный commit?

git add название файла (добафить файл)
git commit --amend -m"..." (изменит последний коммит).


4.Как исправить коммит отправленный на сервер GitHub?

git revert HEAD (изменит последний коммит).


5.Как создать новую ветку и переключится в нее?

git branch название ветки (создаст ветку)

git checkout название ветки (перейдёт в нее)

git checkout -b название ветки (создаст и сразу переключится в нее).


5.В чем отличие между fast-forward merge  and non fast-forfard merge?

fast-forward неявное слияние где коммиты из вливаемой ветки применяются к последнему коммиту целевой ветки,
non fast-forward явное слияние создаёт коммит (два коммита) один последний сливаемой ветки, другой коммит последий целевой ветки.


6.Когда может возникнуть конфликт слияния веток, как  его разрешить?

например при явном слияние появляется ошибка что это невозможно так как на целевой ветке появились новые коммиты, 
это можно исаравить вручную в текстовом редакторе, решив что оставлять а что удалять.

7.Можно ли удалить ветку? Как? 

git branch -d надвание ветки (сначало переклются в другую).
